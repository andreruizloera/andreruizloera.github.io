A field guide

How to Build Great Websites with Claude

The “crazy” designs you screenshot from award galleries aren’t magic. They’re a handful of repeatable techniques. This post walks the stack — and every effect you see here is live, running in your browser, and written from scratch.

June 20, 2026 · 9 min read ClaudeWeb DesignAnimation

I kept sending myself links to sites like motionsites.ai and Awwwards winners thinking how is this even built? After rebuilding a few, the answer is anticlimactic: it’s mostly four or five effects, composed with taste, on a boring-but-solid stack. The hard part isn’t the WebGL — it’s the restraint. Here’s the whole playbook, plus how I actually drive Claude to produce it.

The stack that does the heavy lifting

The marquee above is the first live demo. It’s a few lines of CSS: render the list twice, translate the track by exactly −50%, loop forever. That “seamless infinite scroll” you see on logo walls everywhere — that’s the whole trick.

Effect 1 — Scroll reveals (80% of the “feel”)

If you add one thing, add this. Content fades and rises as it enters the viewport. It costs nothing and instantly reads as “designed.” In plain JS it’s an IntersectionObserver that flips one class: 

const io = new IntersectionObserver((entries) => {
  for (const e of entries) {
    if (e.isIntersecting) { e.target.classList.add('in'); io.unobserve(e.target); }
  }
}, { threshold: 0.2 });
document.querySelectorAll('.reveal').forEach((el) => io.observe(el));

The easing curve matters more than people admit. That cubic-bezier(0.22, 1, 0.36, 1) is an “ease-out-expo” — fast start, gentle settle. Linear easing is what makes amateur animation feel cheap. Every block on this page, including this paragraph, animated in with that curve.

Effect 2 — Word-by-word headline reveals

Split the line into words. Stagger them. Let the easing do the rest.

live demo · staggered text

Split the headline into word spans, then add a stepped transition-delay to each. The hero title at the top of this post uses exactly this. Scroll it back into view and it replays.

Effect 3 — Animated gradient backdrops

The hero at the top of this post is a pure-CSS drifting gradient — a 300%-sized multi-stop background nudged around with one keyframe. No canvas, no shader, runs on a phone from 2018: 

@keyframes gradient-drift {
  0%   { background-position: 0% 50%; }
  50%  { background-position: 100% 50%; }
  100% { background-position: 0% 50%; }
}

People assume these are WebGL. Most of the time they’re four color stops and an animation line. Reach for real shaders only when you need fluid/noise effects CSS can’t fake.

Effect 4 — Pointer-tracking 3D tilt

Move your cursor here

The card tracks your pointer, maps position to a few degrees of rotation, and eases back so it settles instead of snapping.

live demo · hover / move your cursor over the card

The “expensive” feeling here is entirely the smoothing. Without an eased transition the card jitters one-to-one with the mouse and feels broken; with it, the card has weight. That’s the recurring lesson — the physics, not the geometry, sells it.

How I actually drive Claude to build this

The stack is half the battle; prompting is the other half. What consistently works for me:

  • Name the effect, not the vibe. “Add a scroll-reveal with ease-out-expo and a staggered word headline” beats “make it pop.” Claude knows the named techniques.
  • Ask for one reusable primitive at a time. A reveal, a marquee — then compose them. Small pieces are easy to verify and reuse.
  • Demand reduced-motion support up front. One prefers-reduced-motion media query disables every animation for users who need it. It’s an accessibility requirement, not a nicety.
  • Insist on original code. Don’t paste someone’s template and ship it — the effects are generic, the implementations should be yours. Everything on this page was written from the primitives, not copied from a gallery.

The honest takeaway

80% taste, 20% tech.

The techniques are free and everyone has them. What separates a great site is typography, timing, easing, and the discipline to use three effects instead of ten. Claude can write all of it in an afternoon — your job is to art-direct.

Pick one effect from this page, ask Claude for a clean reusable version, and drop it into your next project. Stack a few of those and you’ve got the “how is this built” site. It was never the WebGL.
