How the animated sites are actually built

How to Design Better with AI

The animated sites you screenshot aren’t doing anything exotic. It’s a handful of small effects, all cheap, mostly just CSS. Here’s each one running live, with the code, and how to get AI to build them for you.

June 20, 2026 · 7 min read AnimationCSSClaude

I pulled apart the templates on one of the big motion-design sites to figure out what makes them feel expensive. It comes down to about six effects. None of them needs a designer, and most don’t even need a library. Every demo below is live and built from scratch — view source if you want the real thing.

A drifting gradient background

The most common one. The big gradient at the top of this page is four colors and a single keyframe that slides the background position back and forth. People assume it’s a shader. It isn’t: 

.hero {
  background-image: linear-gradient(120deg, #1f2937, #0e7490, #5b21b6, #1f2937);
  background-size: 300% 300%;          /* bigger than the box, so it has room to move */
  animation: gradient-drift 14s ease infinite;
}
@keyframes gradient-drift {
  0%   { background-position: 0% 50%; }
  50%  { background-position: 100% 50%; }
  100% { background-position: 0% 50%; }
}

The only trick is making the gradient bigger than its container so there’s somewhere to slide, then moving it slowly. Keep it slow — fast gradients look cheap.

Things that fade in as you scroll

Almost every “cinematic” site is mostly this: elements start invisible and slightly low, then fade up when they enter the screen. One IntersectionObserver, one CSS class: 

const io = new IntersectionObserver((entries) => {
  for (const e of entries) {
    if (e.isIntersecting) { e.target.classList.add('in'); io.unobserve(e.target); }
  }
}, { threshold: 0.2 });
document.querySelectorAll('.reveal').forEach((el) => io.observe(el));

The part people get wrong is the easing. That cubic-bezier(0.22, 1, 0.36, 1) — fast to start, slow to land — is most of the difference between this looking deliberate and looking like a slideshow. Every paragraph on this page came in on that curve.

Headlines that reveal word by word

Split the line into words. Reveal them one at a time.

live demo · staggered text

Split the headline into one span per word, give each a slightly later transition-delay, and run the same fade-up. The big title at the top of the page does exactly this. Scroll it off and back and it replays.

An infinite marquee

live demo · infinite marquee

A strip that scrolls forever. The trick is to duplicate the list, then move the track by exactly −50%. When the loop restarts, the second copy is sitting exactly where the first one started, so there’s no visible jump. The fading edges are just a gradient mask on each side.

A card that tilts toward your cursor

Move your cursor over me

I turn your pointer position into a few degrees of rotation, then ease back to flat when you leave.

live demo · hover / move your cursor over the card

Track where the cursor is over the card, map that to a small rotation on two axes, done. The only thing that makes it feel good instead of cheap is easing it back to flat so it settles instead of snapping.

A real 3D object

live demo · 3D object (WebGL) · drag to spin

This is the one that actually impresses people, and it’s the only one here that needs a library. It’s a faceted gem rendered with Three.js, using a glass-like material that bends light (with a little rainbow dispersion along the edges) and a small generated environment so it has something to reflect. It rotates on its own; grab it and throw it. It’s about forty lines of setup. If WebGL isn’t available it quietly falls back to a spinning CSS gem.

Getting AI to build these

You can have Claude write any of these in one shot, but how you ask decides what you get. Vague prompts give vague animations. What works for me:

  • Name the exact effect. “A scroll reveal that fades up with ease-out-expo” gets you something usable. “Make it animated” doesn’t.
  • Ask for one main motion, not ten. Too much movement is the most common way this goes wrong — it stops looking designed and starts looking nervous.
  • Give it numbers. Durations, easing curves, how far things travel. “Fade up 24px over 0.6s” beats “smooth and modern.”
  • Say “respect prefers-reduced-motion.” One media query turns the animation off for people who need it. Easy to forget, worth not forgetting.

Here’s a prompt that builds most of this page’s hero in one go. Expand it, copy it, change the numbers: 

Build a hero section with these animations and nothing else moving:

- Background: a slow gradient drift. Use a large multi-stop gradient
  (background-size around 300%) and animate its position over ~14s. No shader.
- Headline: reveal it word by word on load. Wrap each word in a span and
  fade it up with a small stagger, easing cubic-bezier(0.22, 1, 0.36, 1).
- On scroll: fade the sections below up by 24px as they enter view, same
  easing, once each (IntersectionObserver).
- One accent: a single 3D object with Three.js, slowly rotating, drag to spin.

Keep the motion to just these. Respect prefers-reduced-motion: if it's set,
skip the animations and render the final state.

That’s the whole list

None of these is hard. The effects are free and mostly a few lines each.

What separates the good version from the cheap one is the easing and the restraint: slow curves, and one motion instead of ten. Pick a couple, tune them until they feel right, and stop there.

Grab whichever effect you liked, drop it into your next project, and spend your time on the timing. That is where all the “how is this built” actually lives.
